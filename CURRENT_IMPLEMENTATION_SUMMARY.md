# 当前实现说明

本文档记录当前 `integration/backend-frontend` 分支已经落地的后端、用户端、管理端、数据规范与部署联调内容，便于汇编和 GitHub 交接。

## 项目结构

- `backend/`：Spring Boot 后端，提供 `/api` 与 `/api/v1` 兼容接口。
- `web_user/`：用户端 React + Vite 前端。
- `web_admin/`：管理端 React + Vite 前端。

## 统一接口契约

- 保留统一响应包裹：`{ code, message, data, timestamp }`。
- 保留分页结构：`{ total, list, page, size }`。
- 前端 Axios 适配层会自动解包后端响应，并兼容分页字段到 `items/pageSize`。
- 管理端和用户端均保留 mock 能力，真实联调时通过环境变量或 Vite 代理连接后端。

## 后端已实现内容

- `DesignSpecForumService` 已推进为兼容 facade，Controller 仍保持原调用方式。
- 资源模块已拆分到规范服务：
  - 资源列表、详情、后台查询。
  - 发布、提交、撤回、审核、驳回、下架、版权下架、恢复、删除。
  - 状态变化继续经过 `ResourceStateMachine`。
  - 状态日志和后台操作日志按规则写入。
  - 下载鉴权、下载记录、下载计数和附件摘要返回已接入。
- 求资源模块已接入真实业务：
  - 求资源列表、详情、发布、取消、回复、采纳、后台关闭。
  - 悬赏积分通过 `PointManager` 冻结、返还、转账。
  - 支持免费求资源和积分悬赏，最低积分悬赏为 1 分。
  - 支持状态筛选、积分区间筛选、发布时间/回复量/悬赏积分排序，以及升序/降序。
- 身份与会员模块已迁移：
  - 登录、注册、找回密码验证码、重置密码。
  - 个人资料、修改资料、修改密码、修改邮箱。
  - 登录失败计数、临时锁定、登录日志、JWT 返回结构保留。
  - `/api/me/summary` 和用户资料返回积分、冻结积分、可用积分、悬赏上限等字段。
- 互动与评论模块已迁移：
  - 资源收藏、点赞、评分。
  - 评论列表、详情、新增、回复、修改、删除、点赞。
  - 评分要求成功下载后进行，且同一用户同一资源只能评分一次。
- 通知模块已规范化：
  - `NotificationDispatcher -> NotificationEventService -> NotificationService -> system_notice` 链路落地。
  - 通知事件支持 `PENDING/SENT/FAILED`。
  - 通知失败不回滚主业务事务。
  - 资源状态、评论回复、求资源回复/采纳、会员状态、举报和申诉处理结果可生成通知。
- 管理端后台接口已补齐：
  - `/api/admin/content`
  - `/api/admin/users`
  - `/api/admin/compliance`
  - `/api/admin/catalog`
  - `/api/admin/catalog/options`
  - `/api/admin/config/full`
  - `/api/admin/logs`
  - 以上接口同时保持 `/api/v1/admin/**` 兼容。
- 管理端写操作已接入真实后端：
  - 资源审核、驳回、下架、恢复、版权下架、删除。
  - 求资源关闭、回复删除。
  - 评论隐藏、删除、恢复。
  - 用户启用、禁用。
  - 举报、版权投诉、申诉处理。
  - 分类新增/修改/启用/禁用，标签新增/禁用/合并/一键补齐。
  - 会员等级、积分规则和系统配置更新。
- 分类标签规范已落地：
  - 分类固定为一级、二级两层。
  - 二级分类必须选择启用的一级分类。
  - 同一父级下分类名称不可重复。
  - 禁用一级分类前校验启用子分类。
  - 标签不自由生成，只允许从一级分类名、二级分类名、资源类型中文名组成的规范词库中创建。
- 数据迁移和演示数据已扩展：
  - 管理端演示数据。
  - 用户端真实联调演示数据。
  - 首页、资源库、求资源和分类标签对齐的数据。
  - 规范标签补齐和扩展种子数据。
- 部署基础能力已保留：
  - MySQL + Flyway。
  - 环境变量驱动数据库、JWT、CORS、上传和日志配置。
  - Docker/Compose、部署说明、验收说明和后端部署追踪文档。
  - `application.yml` 已允许用户端 `5173` 与管理端 `5174` 本地联调来源。

## 用户端已实现内容

- 首页推荐分类与规范分类树对齐。
- 资源库支持规范一级分类、二级分类、资源类型、标签、搜索、筛选、排序。
- 求资源列表支持状态筛选、积分区间筛选、发布时间/回复量/悬赏积分排序，以及升序/降序。
- 发布求资源支持：
  - 免费求资源。
  - 积分悬赏。
  - 可用积分、冻结积分、悬赏上限提示。
  - 统一提交 `rewardType/rewardPoints`。
- 求资源详情支持回复、采纳、状态展示和悬赏展示。
- 个人中心支持个人资料、资源、求资源、收藏、点赞、消息、登录记录等汇总数据。
- 资源详情支持附件摘要、评论、下载状态、互动状态和评分规则。
- 前端 mock 数据同步更新为规范分类、标签和资源类型。

## 管理端已实现内容

- 管理端默认连接真实后端；仅 `VITE_ENABLE_MOCKS=true` 时使用 mock。
- 管理端登录接入 `POST /api/auth/login`，只允许 `ADMIN` 角色进入。
- 失效 token 或 401 响应会自动清理登录态并回到登录页，避免半登录状态。
- 所有主要列表页使用后端分页：
  - 内容综合管理。
  - 用户账号管理。
  - 举报版权投诉。
  - 分类标签管理。
  - 操作审计日志。
- 统一分页组件支持首页、上一页、页码、下一页、末页和总数显示。
- 分类标签管理已提供：
  - 新增一级分类。
  - 新增二级分类。
  - 新增规范标签。
  - 一键补齐规范标签。
  - 启用、禁用、合并标签。
- 系统参数配置支持会员等级、积分规则、系统配置保存和缓存刷新。

## 当前未落地或保留延期项

- 不接真实邮件服务。
- 不接 Redis token blacklist。
- 不接对象存储、真实文件流和签名下载 URL。
- 不接 WebSocket、异步队列或邮件/短信推送。
- 不做 DTO 全量替换，当前继续兼容 `Map<String, Object>`。
- Controller 仍保持现有契约，后续可再逐步改为直接调用领域服务。
- 标签仍为全局标签表，暂不增加标签与分类关系表。
